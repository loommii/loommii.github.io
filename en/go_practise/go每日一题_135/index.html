Go Daily Question 135

What is the output of the following code?

package main

import (
    "encoding/json"
    "fmt"
)

type AutoGenerated struct {
    Age   int    `json:"age"`
    Name  string `json:"name"`
    Child []int  `json:"child"`
}

func main() {
    jsonStr1 := `{"age": 14,"name": "potter", "child":[1,2,3]}`
    a := AutoGenerated{}
    json.Unmarshal([]byte(jsonStr1), &a)
    aa := a.Child
    fmt.Println(aa)
    jsonStr2 := `{"age": 12,"name": "potter", "child":[3,4,5,7,8,9]}`
    json.Unmarshal([]byte(jsonStr2), &a)
    fmt.Println(aa)
}
  • A: [1 2 3] [1 2 3];
  • B: [1 2 3] [3 4 5];
  • C: [1 2 3] [3 4 5 6 7 8 9];
  • D: [1 2 3] [3 4 5 0 0 0]

Answer: B Run online

Analysis: As shown in the problem, both print statements use the aa variable.

The content of slice aa and slice a.Child are the same (they point to the same underlying array).

  • Question 1: Why does jsonStr2 Unmarshal modify the content of the aa slice?
  • Question 2: Why is the content of the aa slice not 3,4,5,7,8,9 after Unmarshal runs?

Both questions can actually be answered by looking at the Go documentation:

  • Official (English) https://pkg.go.dev/encoding/json#Unmarshal

To unmarshal a JSON array into a slice, Unmarshal resets the slice’s len to zero and then appends each element sequentially to the slice. As a special case, when unmarshaling an empty JSON array, Unmarshal replaces the existing slice with a new empty slice.

In other words, when the Go JSON library Unmarshal encounters a slice, it essentially does:

a.Child = a.Child[:0] 
a.Child = append(a.Child, 3)
a.Child = append(a.Child, 4)
a.Child = append(a.Child, 5)
a.Child = append(a.Child, 6)
a.Child = append(a.Child, 7)
a.Child = append(a.Child, 8)
a.Child = append(a.Child, 9)

Run online

Question 1: Since aa and a.Child share the same underlying array, they will affect each other.

Question 2: However, when append(a.Child, 6) occurs, a.Child triggers a reallocation mechanism. a.Child then points to a new underlying address, and subsequent appends no longer affect aa.
